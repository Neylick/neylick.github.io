Past Work

"RogueCoon" (C++/Opengl)

Description

A game developped as an indie game-dev, experimenting with creating a game engine featuring full customization freedom while facing lots of challenges.
The game is a simple 2d rogue-like, inspired by Tiny Rogues and Nuclear Throne. The game loop is simple : defeat monsters, gather powerups, clear levels, repeat.

Details

- Stable 144fps (on multiple mid/low-range PCs)
- Optimized OpenGl 2D Raymarching
- Aseprite import pipeline
- Component-based entities & JSON prefabs
- Custom "Immediate UI" (IMUI) (See ImGui )
- TTF Font support and JSON based custom glyphs
- Minimal P2P/Net (via RaspberryPI server)
- Audio (Using fmod)

Asteroids Remake (C++/DX11)

Texturing experiments (Processing)

Description

A Processing (3, based on Java) application used to experiment with normal/height/depth/albedo textures to re-create materials. This application loads a set of maps representing a simple material and projects them onto an object.
I started this project while taking a Graphics Programming/CGI class, while getting more and more curious about the field.
The images represent 2 different materials rendered on a sphere/a plane.

XR Replay for absence in cooperative environment (Unity)

Description

That project was my first introduction to Augmented/Virtual reality (AR/VR) = eXtended Reality (XR).
During a 2 month internship, I created a replay system in an existing client-server architecture project.
This application shows a solution for coworkers working in a VR/AR environment remote that come back from a short abscence and need to catch up.

Details

Features lossless & minimal sized replay saves and a player UI in a cooperative client-server architecture Unity application. The video shows a tech demo for the recording and replaying interface developped.

MRTI : Fruit Ninja Remake (Unity)

Description

A very simple AR game inspired from the Fruit Ninja game series. A tangible objects (card) is placed as a launcher and fruits come out of it. The player must slash the fruits using another object (card, representing a sword), gaining points when successful. They must be careful though : bombs can also appear, making the player end their game after 3 have been cut. Project created with classmates while taking the Mixed reality & tangible interactions (MRTI) elective.
The image represents a prototype for the game, using a phone instead of AR glasses for testing purposes.
